Step 1 — Create the Claude Project

Set up a new Project in Claude.ai and name it correctly before adding any files.

📋 Instructions
  • 1
    Navigate to Claude.ai → Projects
    Click Projects in the left sidebar, then click Create Project.
  • 2
    Name the project
    Use a clear name such as: HOCK International — Content Production Agent
  • 3
    Leave the description optional
    You can add: "AI-managed content production agent for HOCK certification course creation — textbooks, MCQs, slides, scripts, and video."
  • 4
    Do not add any files yet
    Create the project first. You'll add the system prompt, knowledge files, and skills in the next steps in a specific order.
⚠
Important: Add the system prompt before any knowledge files. Claude loads the system prompt first — if it's missing, the agent won't know how to route knowledge file references.

Step 2 — Add the System Prompt

The system prompt is the agent's identity and operating brain. It goes in a dedicated field — not as a knowledge file.

ℹ
File to use: HOCK_System_Prompt.md — paste the full contents into the Project Instructions field.
📋 Instructions
  • 1
    Open Project Settings
    Inside your new project, click the Settings or Edit Project option (gear icon or pencil icon depending on your Claude version).
  • 2
    Find the "Project Instructions" field
    This is the system prompt field — it may be labeled "Instructions," "Custom Instructions," or "System Prompt" depending on your Claude plan tier.
  • 3
    Open HOCK_System_Prompt.md and copy the full contents
    Open the file in any text editor or markdown viewer. Select all (Ctrl+A / Cmd+A) and copy.
  • 4
    Paste into the Project Instructions field
    Paste the entire contents. Do not summarize, truncate, or reformat — paste it exactly as written.
  • 5
    Save the project settings
    Click Save or Done. The system prompt is now active for every conversation in this project.
🧠 What's in the System Prompt
Agent identity Production order (locked) Core operating rules Workflow state rule Session state tracking Skills invocation map Phase transition rule Response style External tool references

Step 3 — Add Knowledge Files

Three knowledge files go into Project Knowledge. These load into every conversation and are cross-referenced by the system prompt.

⚠
Order matters: Upload Core Operations first, then Content Standards, then Revision Prompts. This matches the reference order in the system prompt.
📁 File 1 — HOCK_Core_Operations.md
  • 1
    Go to Project Knowledge
    Inside the project, find the Knowledge or Project Files section. Click Add Content or the upload button.
  • 2
    Upload HOCK_Core_Operations.md
    Upload the file directly — do not paste as text. Uploading as a file preserves the name, which the system prompt references.
  • 3
    Confirm the file appears in the knowledge list
    You should see it listed with the filename intact. If it was renamed during upload, rename it back to HOCK_Core_Operations.md.
Phase Router Workflow State Table Phase Behavior Connector Webhooks Artifact Naming Intent Labels
📁 File 2 — HOCK_Content_Standards.md

Upload the same way. This is a quick-reference summary — the skills are still the authoritative source, but this gives Claude fast in-context lookup for depth tables, style rules, and MCQ specs.

Depth Calibration Matrix Textbook Style Rules MCQ Format Token Chunking Tables
📁 File 3 — HOCK_Revision_Prompts.md

Upload last. This file is large by design — it contains all 20+ revision prompt templates used during live content editing. Do not trim or summarize this file.

Textbook Revisions (5 types) MCQ Revisions (5 types) Slide Outline Revisions (6 types) Script Revisions (5 types)

Step 4 — Upload Skills

Skills are uploaded to Project Knowledge the same way as knowledge files, but they function differently — Claude invokes them by name as active instruction sets during content generation.

ℹ
How skills work: When Claude sees invoke `skill-name` in its instructions, it looks up the matching file in Project Knowledge by its name: frontmatter field — not the filename. The name in the frontmatter header must match exactly what's in the system prompt.
⬆ Upload Order & Names
# File to Upload Frontmatter Name (must match) Priority
1 Depth_Calibration_Skill.md depth-calibration-skill Always First
2 Token_Optimization.md token-optimization Always Second
3 Textbook_Style_Guide.md textbook-style-guide Phase 1
4 MCQ_Creation_Skill.md mcq-creation-skill Phase 2
5 slide-creation-SKILL.md slide-creation-skill Phase 3 — Presentation Outline
6 slide-script-SKILL.md slide-script-skill Phase 5 — Slide Script
7 monday-workflow-SKILL.md monday-workflow-skill Phase 1 Required Other Phases User-Directed
⚠ Skills vs Knowledge Files — Key Difference

Knowledge Files

  • Always loaded into context
  • Used for reference and lookup
  • Claude reads passively
  • Supports decisions and routing

Skills

  • Invoked by name when needed
  • Active instruction sets
  • Drive content generation behavior
  • Must have matching frontmatter name

Step 5 — Verify the Setup

Run these test prompts in the project before handing it off to a HOCK user. Each test confirms a different layer of the agent is working correctly.

✅ Test 1 — Identity & Phase Awareness
# Paste this into a new conversation inside the project What phase are we in and what are you waiting for before we can begin?

Expected: Agent identifies it's in Initiation phase, states it needs source materials uploaded and a textbook prompt before proceeding. It should NOT start generating content.

✅ Test 2 — Depth Calibration Trigger
I want to start the textbook for CIA Part 1, Domain 1 — Foundations of Internal Auditing.

Expected: Agent invokes depth-calibration-skill, asks for syllabus or guides you through scope/cognitive level/exam weight questions. It should NOT generate content yet.

✅ Test 3 — Phase Lock
Skip the textbook and just generate MCQs for me now.

Expected: Agent blocks the request, explains MCQs require an approved textbook first, and asks the correct gating question to get back on track.

✅ Test 4 — Monday Skill Boundary
Update Monday after this is done.

Expected: For Phase 1 textbook, agent automatically triggers the Monday update after finalization — asks Word Doc or Markdown, then posts to the correct field. For all other phases, agent confirms it will only update Monday when explicitly instructed.

🔧 If Something Doesn't Behave Right
  • 1
    Agent doesn't recognize its phase rules
    Check that HOCK_Core_Operations.md was uploaded and the filename wasn't changed. The system prompt references it by name.
  • 2
    Agent skips depth calibration
    Confirm the Depth_Calibration_Skill.md frontmatter says name: depth-calibration-skill exactly. Even one character difference breaks the invocation.
  • 3
    Agent ignores phase locking and generates out of order
    Re-check that the system prompt was saved in the Project Instructions field — not uploaded as a knowledge file.
