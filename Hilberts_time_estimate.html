Workflow 1 — Email Triage & Extraction
The first automation: n8n watches the Outlook inbox, Claude classifies every incoming email, and all ITBs are extracted into structured bid records.
Phase 1 · Target: Days 1–5
✓
Delivers standalone value. Workflow 1 runs independently — every ITB is captured and structured even before drawing retrieval is built.
TaskHours
n8n O365 Watch Emails trigger + polling configuration
15-minute polling interval. Configure to capture subject, body, sender, attachments, and timestamp.
2 hrs
Claude classification prompt (ITB vs. non-ITB)
Haiku model — fast and low cost. Returns is_itb, confidence level, and reason. Routes non-ITBs to stop.
3 hrs
Router node — confidence-based branching
High/medium confidence → extraction. Low confidence → flagged tab. False → stop. No email is silently dropped.
2 hrs
Claude extraction prompt — 14-field structured output
Sonnet model. Extracts project name, GC, contact, bid due date, drawing links, format type, portal credentials, scope keywords, and HVAC scope summary.
4 hrs
Google Sheets append — bid register write
Parse Claude JSON, map all 14 fields to sheet columns, add workflow metadata (received_at, status, source_email_id).
3 hrs
Error handling — invalid JSON, API failures, rate limits
Wrap Claude calls in try/catch. Log failures to an Errors tab rather than crashing the scenario.
2 hrs
Live test on 5 real ITB samples
Run against the collected samples. Verify classification accuracy, field extraction quality, and Sheets write.
2 hrs

Phase 1 · Step 3 of 5
Workflow 2 — Drawing Retrieval
The highest-value workflow. Routes each bid to one of three retrieval paths based on drawing format — PDF attachments, Dropbox bulk packages, or contractor portals.
Phase 1 · Target: Days 4–12
TaskHours
Trigger: Sheets watch — new "Awaiting Drawing Retrieval" row
n8n watches for rows with the correct status. Routes to the correct path based on drawing_format field.
2 hrs
Path A — PDF attachment: Claude Vision page identification
O365 retrieve attachment binary → Claude Vision identifies HVAC sheet pages (M-, H-, HVAC- prefixes) → returns page numbers, equipment types, and scope summary.
6 hrs
Path B — Dropbox: folder listing and fuzzy name matching
Dropbox list_folder on shared link → Claude matches contractor bid names to Dropbox folder names (handles "Dupont vs Nemours" style mismatches) → navigate matched folder → identify HVAC files.
8 hrs
Path B — Dropbox: generate direct file links
Dropbox create_shared_link on matched HVAC files → write direct links to Sheets row. High-confidence matches only; low-confidence flagged for manual review.
3 hrs
Path C — Portal links: flag and surface credentials
Portal URLs, Project IDs, and Security Keys already extracted in W1. Update row status to "Portal — Manual Access" with all credentials visible in the morning card.
2 hrs
Sheets row update — status to "Ready for Review"
All paths write their results back. Rows transition from "Awaiting Drawing Retrieval" to "Ready for Review" when complete.
2 hrs
Live test — all three formats against real samples
Verify PDF Vision accuracy, Dropbox fuzzy match on at least 3 real naming pairs, portal flag behavior.

Workflow 3 — Morning Review Digest
The final workflow: a daily 6:45am email to Tom with all pending bids organized by urgency, drawing links ready, and one-click BID / PASS / FLAG decision buttons.
Phase 1 · Target: Days 10–15
TaskHours
n8n scheduled trigger — 6:45am daily
Schedule node reads all "Ready for Review" rows from the Sheets register, sorted by bid_due_date ascending.
1 hr
Claude digest assembly — HTML email generation
Claude receives the bid array and produces a clean HTML email. Bids sorted by due date. Urgent badge (red) if due within 48 hours. Each card shows: project, GC, due date, scope summary, drawing link, and action buttons.
5 hrs
O365 send — morning email to Tom (+ CC Belinda)
Subject: "Bid Queue — {N} new · {M} due this week". Sent from the automation at 6:45am daily.
1 hr
Webhook setup — BID / PASS / FLAG decision capture
Three n8n webhook URLs. Each button in the digest email hits the correct webhook with the row ID encoded. Catches Tom's click instantly.
4 hrs
Webhook receiver — Sheets update + tab routing
BID → status = "Bidding", row copies to Active Bids tab. PASS → Passed tab with timestamp. FLAG → stays in Pending, highlighted. All decisions logged with decision_at timestamp.
3 hrs

Send test digest to Jessica and Marc first. Review card layout, link behavior, and webhook response before live delivery.
