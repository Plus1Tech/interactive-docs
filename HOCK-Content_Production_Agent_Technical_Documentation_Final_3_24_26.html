MCQ Creation

Scenario-based exam questions generated directly from the approved textbook. Reviewed in batches, then saved to Monday and automatically converted to an Excel file for LearnDash import.

Requires approved textbook
What Makes a Quality MCQ
✓

Forces reasoning, not just recall

Candidates must work through the scenario to eliminate wrong answers — not just recognize a keyword.

✓

All four answer options are similar in length

A noticeably longer correct answer tips off the candidate. Every option is balanced before the batch is finalized.

✓

Each wrong answer has a teaching explanation

All four options get a full explanation — not just the correct one. These are learning moments, not just feedback.

🚫

No absolute language anywhere

Words like "always," "never," and "exclusively" make wrong answers obvious and are prohibited.

How Many Questions Per Study Unit?
Depends on the depth level assigned during calibration
🟢

Basic depth → 8 to 12 questions

Recall-level difficulty. Narrow topic, lower exam weight.

🟡

Medium depth → 15 to 20 questions

Apply and analyze. Moderate topic with balanced exam weight.

🔴

Comprehensive → 20 to 30 questions

Evaluate and synthesize. Broad topic with high exam weight.

MCQ Creation — Implementation Rules

Phase 2. Requires confirmed approved textbook. Output is a raw JSON array only — no markdown fences, no plain text, no labels. First batch always 15 questions.

First batch: 15Raw JSON array output only
Question Construction Rules
1

Choice A is always the correct answer

B, C, D are plausible distractors. This is fixed — never randomized.

2

All four choices must be similar in length

Compare lengths after writing. Rewrite if uneven — a longer correct answer signals it.

3

No extreme words anywhere in choices

Banned: always, never, exclusively, only, solely, under no circumstances.

4

Scenario-based — candidates must reason to eliminate

Not recall-based. Each question should require applying knowledge to a scenario.

5

Never reference "the content," "the material," or "the textbook"

Strictly prohibited in all question and choice text.

6

All four choices get a full explanation

50–100+ words each. No "This choice is correct/incorrect because" opener — state the reasoning directly.

Return ONLY a raw JSON array. No fences, no text before or after, no labels.

Content Standards

Every deliverable is calibrated to the right depth before production begins. Depth is determined by three factors — how many concepts are covered, what level of thinking the exam expects, and how much the topic is weighted on the exam.

Calibration required before every run

Content Voice & Writing Standards
🎯

Who the content is written for

Professionals with 2–4 years of experience studying for a specific certification. The tone is senior explaining to junior — direct, intelligent, and assumes capability without assuming prior knowledge of the specific topic.

📐

How the content is structured

65% narrative prose, 25% bullet lists, 10% examples and tables. No hedging language, no filler transitions, no AI writing patterns. Key terms are bolded on first mention only. Standards are cited by number and woven into prose naturally.

Content Standards — Implementation Reference

Depth calibration runs before every content type. Three required inputs: scope (concept count), cognitive level (highest action verb from the syllabus sub-item), and exam weight (% for this section). Confirmed with user before generation begins.

Calibrate before every generation
Depth Calibration — 3 Inputs
Step 1 — Scope
Count distinct concepts in the syllabus line item
Narrow: 1–2 conceptsModerate: 3–5Broad: 6+
Step 2 — Cognitive Level
Use action verb from the specific sub-item (not parent). If multiple, use highest.
Basic: define, list, describeMedium: apply, compare, assessComp: evaluate, design, integrate
Step 3 — Exam Weight
From syllabus. If not explicit, infer from section position.
Low: under 10%Medium: 10–20%High: over 20%
Textbook Generation Structure
One prompt = one complete Study Unit (all lettered sub-sections)
A

Exam Introduction — first Study Unit only

Exam name, duration, question count, all sections with % weights (read from syllabus). Written once — never repeated.

B

Study Unit Introduction — every Study Unit

# Study Unit [N]: [Title] (H1). 3–4 prose paragraphs. Connects all lettered sub-sections.

C

First lettered sub-section → PAUSE

## Na. Title (H2 matching syllabus exactly). Prose, H3 concept headings, examples. Stop and ask before continuing.

D

Remaining sub-sections on YES

All remaining lettered sub-sections in sequence. ONE Summary Table (2-column: Key Concept | Summary) after the final sub-section only — never after each sub-section.
