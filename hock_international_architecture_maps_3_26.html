System Overview
The HOCK Content Production Agent is a 7-phase AI workflow that generates exam prep materials end-to-end — from raw textbook to final slide script. Claude generates content; the agent manages orchestration, routing, and storage.

🧱
Core design principle: Claude generates content. The agent approves, routes, saves, and tracks. These responsibilities never cross.

Division of Responsibility

🧠 Claude — Content Engine
Generates textbook sections, MCQs, slide outlines, and narration scripts. Operates inside a Claude Project with certification-specific knowledge files. Does not manage state, approvals, or file attachments.
⚙️ n8n Agent — Workflow Engine
Manages phase locking, approval gates, Dropbox saves, Monday.com status updates, Gamma builds, Kevin subtask creation, and all file attachment operations.

Session State & Key IDs
How session context is maintained, what identifiers are used, and how the agent tracks state.
🔗
Identity rule: sessionId = taskName = textbookName. These three always hold the same value throughout the session.

Session Variables

VariableValue SourceNotes
sessionIdUser provides at session startEquals taskName and textbookName throughout
mondayItemIdAgent retrieves via search_taskNever ask user — always look up
dueDateAgent passes explicitlyNever computed by n8n
gammaGenerationIdReturned by Gamma MCP on creationStored immediately, never lost, used for polling and PPTX export
studyUnitIdFrom session contextUsed in file naming: SU_5_2796 format
phaseStateWindow Buffer Memory (n8n)Required for phase context — removing causes agent looping

n8n Architecture Notes

  • Tool nodes only pass data to sub-workflows if input fields are explicitly defined on the tool node
  • Window Buffer Memory node is required — removing it causes the agent to loop and consume API rate limits
  • Tool descriptions must never reference field names not defined in the tool's input schema
  • Behavioral rules belong in the system prompt, not tool descriptions
  • n8n Code node does not support special characters (e.g. →) in template literals — use string concatenation
  • Define Below source option removes sub-workflow selection — use Database source instead

Agent Hard Rules

  • Never ask user for Monday Task ID — always retrieve via search_task
  • Never create a new Gamma presentation if a gammaGenerationId already exists for the session
  • Never offer "Markdown vs Doc" save options — these don't exist in the defined workflow
  • Always attempt Path A (base64 via file_base64) for MCQ Excel upload first
  • Never batch Monday MCP updates — each field update is a separate call
  • Claude never creates binary files — file conversion happens in n8n
  • Dropbox saves are always manual user actions — never automated
